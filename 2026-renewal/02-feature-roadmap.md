# QUIPU 2026 Renewal Feature Roadmap

## 1. [Must] 백오피스 인증 및 권한 체계 고도화
- 담당자: 어진원
- 문서: [01-bo-auth.md](./feature/01-bo-auth.md)

백오피스 인증을 Google OAuth 기반으로 전환하고, 운영 권한을 역할 단위로 분리한다.

- 로그인 방식: Google OAuth 단일화
- 권한 레벨: `read/all`, `write/all`, `write/activity`, `write/recruit-form` 등 라벨 기반 역할 분리
- 슈퍼어드민 정책
  - 슈퍼어드민 계정(예: QUIPU 공식 Google 계정)은 고정
  - 슈퍼어드민만 사용자 계정 등록 및 권한 부여/변경 가능
- 접근 정책
  - 사전 등록되지 않은 Google 계정은 백오피스 접근 불가

## 2. [Must] 메인 웹 Activity의 동적 구성 기능 도입
- 담당자: 정재민
- 문서: [02-activity-cms.md](./feature/02-activity-cms.md)

메인 웹 Activity를 정적 코드 의존 구조에서 DB 조회 기반 구조로 전환한다.

- 메인 웹은 Activity 데이터를 DB에서 조회하여 렌더링
- 백오피스에서 Activity 구성 정보를 직접 관리
  - `type` 관리 (예: `study`, `semina` 등)
  - 각 type별 `field` 관리 (예: `date`, `title`, `content`, `img` 등)
  - 각 field별 `content` 관리 (예: `24.03.01`, `깃의 원리에 대해서` 등)
- 백오피스 설정 결과가 메인 웹 화면에 일관되게 동적으로 반영되어야 함

## 3. [Must] 동아리 공통 정보 관리의 백오피스 일원화
- 담당자: 최재혁, 이건희
- 문서: [03-club-info-cms.md](./feature/03-club-info-cms.md)

동아리 핵심 정보를 백오피스에서 관리하고, 메인 웹의 필요한 곳곳에서 해당 데이터를 조회하여 렌더링한다.

- 백오피스 관리 대상
  - 임원진 정보: 이름, 포지션, 학과, 학번, 전화번호, 인터뷰
  - 대외 링크: 홈페이지, 인스타그램, GitHub, 이메일
  - FAQ
  - 등등
- 메인 웹 반영 대상
  - Interview 영역
  - Footer 영역
  - FAQ 관련 노출 영역
  - 등등

## 4. [Must] 모집 폼의 동적 구성 기능 도입
- 담당자: 오형규, 김민건
- 문서: [04-recruit-form-cms.md](./feature/04-recruit-form-cms.md)

모집 폼을 고정 양식에서 벗어나, 백오피스에서 동적으로 설계·운영 가능한 구조로 전환한다.

- 백오피스에서 모집 폼 항목/구성 직접 설정
- 메인 웹은 활성화된 폼 정의를 기반으로 지원 화면 렌더링
- 연도별 모집 정책 변경 시 코드 수정 없이 설정 변경만으로 반영 가능해야 함
- 구글폼 참고

## 5. [Next] 동아리 코멘트 게시판 기능 추가
- 담당자: 미정
- 문서: [05-comment-board.md](./feature/05-comment-board.md)

메인 웹에서 누구나 한 줄 코멘트를 제출할 수 있고, 백오피스 승인 후 공개되는 게시판 기능을 도입한다.

- 코멘트 데이터 구조
  - `content`: 코멘트 내용
  - `author`: 작성자 이름 자유 기재(예: `컴맹`, `전전컴20`, `코딩천재`, `홍길동`)
  - `timestamp`: 등록 시각
- 운영 정책
  - 메인 웹에서 누구나 로그인 없이 코멘트 등록 가능
  - 등록 즉시 공개하지 않고 승인 대기 상태로 저장
  - 백오피스에서 승인/거절/삭제 처리
  - 승인된 코멘트만 메인 웹에 노출
